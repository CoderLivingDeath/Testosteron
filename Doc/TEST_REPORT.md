# Отчет о ручном тестировании системы «Testosteron»

**Дата тестирования:** 17 февраля 2025 г.  
**Версия системы:** 1.0.0  
**Тестировщик:** QA-команда  
**Статус:** Тестирование завершено

---

## 1. Введение

### 1.1 Цель тестирования

Проверка работоспособности всех функций системы онлайн-тестирования «Testosteron» в соответствии с техническим заданием (TR_Template.md) и отчетом о разработке (Report.md).

### 1.2 Область тестирования

- Публичная часть сайта (главная страница, список тестов, прохождение тестов)
- Аутентификация и авторизация (регистрация, вход, выход)
- Административная панель (создание, редактирование, удаление тестов)
- Безопасность приложения
- Кроссбраузерная совместимость

### 1.3 Окружение тестирования

| Параметр | Значение |
|----------|---------|
| Браузер | Google Chrome 121.0.6167.85 |
| ОС | Windows 11 Pro 22H2 |
| Разрешение | 1920×1080 |
| СУБД | PostgreSQL 16.1 |
| .NET SDK | .NET 10.0 |
| Тестовый сервер | localhost:5000 |

## 5. Рекомендации

### 5.1 По исправлению дефектов

1. **Приоритет 1 (Критические)** - исправить до релиза:
   - BUG-003: Добавить серверную валидацию обязательных полей
   - BUG-004: Реализовать HTML5-валидацию или клиентскую проверку

2. **Приоритет 2 (Важные)** - исправить перед выпуском:
   - BUG-005: Добавить валидацию названия теста
   - BUG-001: Исправить адаптивность меню на мобильных
   - BUG-002: Добавить сообщение "Тесты не найдены"

3. **Приоритет 3 (Низкие)** - исправить в следующих спринтах:
   - BUG-006: Добавить проверку максимальной длины
   - SEC-001: Реализовать rate limiting

### 5.2 По улучшению функционала

1. Добавить drag-and-drop сортировку полей в редакторе тестов
2. Реализовать функционал восстановления пароля
3. Добавить блокировку учетной записи после 5 неудачных попыток

### 5.3 По тестированию

1. Провести повторное тестирование после исправления дефектов
2. Добавить автоматизированные тесты для критических сценариев
3. Расширить тестирование безопасности (нагрузочное тестирование)
