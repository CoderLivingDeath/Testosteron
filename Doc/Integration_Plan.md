## 4. ТЕХНИЧЕСКОЕ ЗАДАНИЕ

### 4.1 Цель автоматизации

Автоматизация процессов создания, управления и прохождения онлайн-тестирований с гибкой структурой вопросов и возможностью анализа результатов.

### 4.2 Наименование и краткая характеристика системы

**Наименование:** Система онлайн-тестирования «Testosteron»

**Краткая характеристика:**
Веб-приложение на базе ASP.NET Core MVC с использованием PostgreSQL для хранения данных. Система обеспечивает создание тестов с различными типами вопросов, прохождение тестов по уникальным ссылкам, хранение и анализ результатов.

**Тип системы:** Веб-приложение (MVC)
**Архитектура:** Трехуровневая (Presentation → Business Logic → Data Access)

### 4.3 Назначение и функции предмета разработки

#### 4.3.1 Назначение системы

Система предназначена для:

- Создания и управления тестами с различными типами вопросов
- Прохождения тестов пользователями через веб-интерфейс
- Хранения и отображения результатов тестирований
- Аутентификации и авторизации пользователей

#### 4.3.2 Функции системы

| ID | Функция | Описание | Приоритет |
|----|---------|----------|-----------|
| F001 | Аутентификация | Вход в систему по логину и паролю | Высокий |
| F002 | Регистрация | Создание новой учетной записи | Средний |
| F003 | Просмотр списка тестов | Отображение доступных тестов | Средний |
| F004 | Прохождение теста | Заполнение и отправка ответов | Высокий |
| F005 | Создание теста | Создание нового теста с полями | Высокий |
| F006 | Редактирование теста | Изменение полей и параметров теста | Высокий |
| F007 | Удаление теста | Удаление теста и связанных данных | Средний |
| F008 | Просмотр результатов | Просмотр своих результатов тестирований | Средний |
| F009 | Предпросмотр теста | Просмотр теста перед публикацией | Средний |

#### 4.3.3 Типы вопросов

| Тип | Описание | UI-компонент |
|-----|----------|--------------|
| text | Текстовый ответ | Textarea |
| radio | Выбор одного варианта | Radio buttons |
| check | Выбор нескольких вариантов | Checkboxes |

### 4.4 Требования к предмету разработки

#### 4.4.1 Функциональные требования

**Аутентификация и авторизация:**

- Регистрация пользователей с валидацией email
- Вход по логину и паролю
- Ролевая модель (admin, user, guest)
- Защита административной области

**Управление тестами:**

- Создание тестов с названием (до 200 символов) и описанием (до 600 символов)
- Добавление полей вопросов с типами text/radio/check
- Редактирование и удаление тестов
- Предпросмотр тестов

**Прохождение тестов:**

- Отображение тестов по уникальным ссылкам (GUID)
- Поддержка трех типов полей
- Валидация обязательных полей
- Отправка ответов через AJAX
- Сохранение ответов для зарегистрированных и гостевых пользователей
- Проверка на повторное прохождение

#### 4.4.2 Требования к надежности
